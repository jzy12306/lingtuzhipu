请按照以下步骤手动修复 Docker 镜像配置：

1. 关闭 Docker Desktop（右键系统托盘图标，选择 Quit Docker Desktop）

2. 编辑或创建以下文件：
   C:\Users\[你的用户名]\.docker\daemon.json

3. 使用以下内容（清空所有 registry-mirrors 配置）：

{
  "builder": {
    "gc": {
      "defaultKeepStorage": "20GB",
      "enabled": true
    }
  },
  "experimental": false,
  "features": {
    "buildkit": true
  },
  "registry-mirrors": []
}

4. 保存文件并重新启动 Docker Desktop

5. 重启后，运行以下命令测试：
   docker pull mongo:7
   docker pull redis:7-alpine

6. 如果这些命令成功，再回到本项目目录运行：
   docker-compose -f docker-compose-simple.yml up -d

或直接尝试完整部署：
   docker-compose up -d --build
